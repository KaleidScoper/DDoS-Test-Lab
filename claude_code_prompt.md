## 项目背景（必读，用于理解数据含义）

论文题目：基于仿真的 DoS/DDoS 攻击检测与缓解策略研究

实验平台：
- 靶机：树莓派 4B，IP 192.168.1.111，运行 Nginx，千兆网卡
- 攻击端：树莓派 5，通过 Docker macvlan 部署容器，每容器有独立 IP
- 合法客户端容器：20 个，IP 段 192.168.1.128–192.168.1.147
- 攻击者容器：最多 50 个，IP 段 192.168.1.150–192.168.1.199
- 局域网段：192.168.1.0/24

防御系统三层架构：
1. **CUSUM 全局攻击态检测**：监测入站总包速率 xn，递推 Sn = max(0, Sn-1 + xn − μ0 − k)，参数 μ0=120pps，σ0=35pps，k=3σ0=105pps，h=5σ0=175pps，冷却窗口30秒。Sn 超过 h 时进入攻击态，档位边界收紧（白名单门槛80→90，普通档门槛50→60，可疑档门槛20→30）。
2. **IP 信誉动态维护**：每IP分数[0,100]，初始50。指数衰减半衰期300秒向50回归。行为奖惩：合规+0.5/s，轻度违规-2/s，严重违规-10/s，SYN Flood特征附加-5/s。
3. **nftables 四档分级限速**（正常态/攻击态下门槛不同）：
   - 白名单 (≥80/≥90)：直接 accept，不限速
   - 普通档 (50–79/60–89)：限速 225pps（攻击态收紧至 190pps）
   - 可疑档 (20–49/30–59)：限速 155pps
   - 黑名单 (<20/<30)：限速 10pps（实际接近全丢）

关键实验数据（模拟数据必须对齐这些数值）：

**基线（场景一）**：
- 总入站 pps 均值 120，标准差 35
- 入站流量约 0.075 Mbps
- CPU 使用率约 50%（单核等效）
- 网卡层丢包率接近 0（约 0.5pps）
- 合法客户端成功率 100%
- CUSUM Sn ≈ 0

**UDP Flood 无防御（场景二）**：
- 总入站 pps 均值 300,000，峰值近 400,000
- 入站流量均值 134 Mbps
- CPU 约 65%（单核等效）
- 网卡层丢包均值 206pps，峰值 348pps
- 合法客户端成功率约 4%

**SYN Flood 无防御（场景二）**：
- 总入站 pps 均值 163,000，峰值 175,000
- 入站流量均值 74.5 Mbps，峰值 79.9 Mbps
- 网卡层丢包均值 45,500pps，峰值 88,500pps（SYN backlog 满后内核丢弃）
- 合法客户端握手成功率约 2%

**UDP Flood 有防御（场景三）**：
- CUSUM 告警响应时间：攻击开始后约 3.2 秒触发
- 拦截率：稳态（t>60s）约 99.5%（黑名单贡献约 95%，限速贡献约 4.5%）
- 合法吞吐恢复：防御生效后约 15 秒内从 4% 恢复至 97%
- CUSUM Sn 在攻击期间持续上升至约 1.8×10⁸ 量级，攻击停止后快速归零
- 攻击者 IP 约 30 秒内全部进入黑名单档
- CPU 稳态约 50%

**SYN Flood 有防御（场景三）**：
- CUSUM 告警：约 4.5 秒（SYN Flood 总 pps 较低，累积慢）
- 拦截率：约 99.2%
- 合法吞吐恢复至 94%

## 功能需求

### 整体结构
单页面应用，左侧固定侧栏（导航/操作面板），右侧主内容区（图表+状态）。

### 导航/阶段控制（左侧栏顶部）
用户可按顺序推进或跳转以下实验阶段，当前阶段高亮：
1. **系统就绪** — 初始状态，防御系统在线，无攻击
2. **基线采集** — 仅合法客户端运行，采集 10 分钟基线（模拟快进，实际30秒内完成动画）
3. **攻击类型选择** — 提供 UDP Flood / SYN Flood 两个按钮
4. **关闭防御 / 启动攻击** — 无防御场景
5. **开启防御** — 三层防御激活，实时展示 CUSUM 告警、信誉变化、拦截效果
6. **消融对比** — 静态展示消融实验柱状图
7. **NS-3 扩展性** — 静态展示 NS-3 折线图

### 左侧栏参数面板


## 数据模拟逻辑

使用 `setInterval` 每 1000ms 推进一个仿真 tick。根据当前阶段生成以下数据：

**基线阶段**：
- total_pps：以 N(120, 35²) 采样，加轻微自相关噪声（AR(1) 系数 0.3）
- pass_pps = total_pps（无拦截）
- CPU：约 50% + 小幅波动
- 合法成功率：100% + 微小抖动

**攻击无防御阶段（UDP）**：
- 攻击开始后 total_pps 在约 10 秒内线性爬升至 300,000，后保持在 280,000–400,000 之间随机游走
- pass_pps = total_pps（无拦截）
- CPU 爬升至 60–70%
- 合法成功率骤降至 2–6%

**攻击无防御阶段（SYN）**：
- total_pps 爬升至 163,000，波动范围 150,000–175,000
- 网卡层丢包高（45,000–88,000pps），合法成功率降至 1–3%

**攻击有防御阶段**：
- 攻击开始：total_pps 同无防御
- pass_pps 初期与 total_pps 相同（防御未感知）
- CUSUM Sn 每 tick 按公式累积
- 约 3–4 秒后（UDP）或 4–5 秒后（SYN）Sn 超过 h，进入攻击态
- 进入攻击态后：各攻击者 IP 信誉按 -10/tick 快速下降，约 30 秒内全部进入黑名单
- pass_pps 随黑名单 IP 增加而迅速下降：稳态下 pass_pps ≈ total_pps × (1 - 0.995)
- 合法成功率约 15 秒后从 4% 回升至 97%
- CUSUM Sn 累积至约 1.8e8 后保持高位（攻击持续中），攻击停止后 18 秒内归零

信誉状态机（每 tick，针对 70 个 IP 分别维护）：
- IP .128–.147（合法）：初始信誉 50，每 tick +0.5，缓慢升至 80+进入白名单
- IP .150–.199（攻击者）：基线阶段不活跃（信誉 50），攻击阶段每 tick -10（UDP严重违规）或 -15（SYN Flood附加惩罚），信誉快速降至<20进入黑名单

## 页面布局与视觉风格

**禁止**：圆角、渐变色、emoji、阴影（除低调的 box-shadow: inset 类分隔线）、动画过渡（除图表数据更新）。

**允许**：实线边框、亚克力效果（backdrop-filter: blur，背景半透明，用于左侧栏与图表信息框）、细分割线。

**配色方案**（仿 VS Code Light / GitHub Light）：
```css
--bg:          #ffffff
--bg-panel:    rgba(248, 248, 248, 0.92)
--border:      #e1e4e8
--text:        #24292e
--text-muted:  #6a737d
--text-code:   #005cc5
--accent:      #0366d6
--danger:      #d73a49
--success:     #22863a
--warning:     #e36209
```

**字体**：
- UI 文字：`'Segoe UI', 'SF Pro Text', system-ui, sans-serif`
- 数值/参数：`'Cascadia Code', 'Fira Code', 'Consolas', monospace`

**布局**：
- 左侧导航栏：固定宽度 260px，全高，背景 `--bg-panel`，左边界 `--border`
- 主内容区：剩余宽度，grid 布局，图表自适应排列

**信息面板**（左侧栏顶部，描述当前阶段）：
简洁一段话，不超过 3 行，说明当前正在演示什么，字号小，颜色 `--text-muted`。

## 操作按钮说明

左侧栏按钮区域（各阶段对应按钮，非当前阶段可点击但灰色提示顺序）：

- **[开始基线采集]**：启动基线阶段模拟（30秒快进动画，完成后自动计算 μ0/σ0 并更新参数面板）
- **[选择攻击类型]**：UDP Flood / SYN Flood 两个子按钮
- **[关闭防御 → 启动攻击]**：无防御攻击场景，持续运行直到手动切换
- **[启动三层防御]**：在当前攻击场景中激活防御
- **[停止攻击]**：攻击者停止发包，观察 CUSUM Sn 归零过程
- **[消融实验对比]**：切换到消融实验图表视图
- **[NS-3 扩展性实验]**：切换到 NS-3 图表视图
- **[重置实验]**：清空所有数据，回到初始状态
